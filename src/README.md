# Estructura de carpetas y patrones

Este proyecto sigue una arquitectura escalable y mantenible basada en buenas prácticas de React:

- **components/**: Componentes de UI y contenedores, organizados por dominio.
- **hooks/**: Hooks personalizados reutilizables.
- **services/**: Lógica de acceso a servicios externos (ej: Supabase, APIs).
- **contexts/**: Contextos globales de React para estado compartido.
- **constants/**: Valores constantes y configuraciones globales.
- **styles/**: Archivos CSS y utilidades de estilos.
- **assets/**: Imágenes, íconos y recursos estáticos.

## Patrones aplicados
- Hooks personalizados
- Context API para estado global
- Separación de componentes presentacionales y contenedores
- Centralización de servicios
- Uso de constantes y estilos globales

Lee los README de cada carpeta para más detalles.
